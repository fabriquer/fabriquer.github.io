Fabriquer style guide

Who doesn't love a style guide?

Requirements

Everything in this list is here for a reason. Changes to these requirements should only be made for an even better reason (not "it conflicts with my personal aesthetic").

  1. Compile with -Weverything
    • Aside from the whitelisted warnings below, Fabriquer should always build with all warnings. With Clang, -Wall gives a lot of warnings but -Weverything actually produces all the warnings.
    • Some of these warnings seem silly (who cares if a constructor takes a parameter with the same name as a member variable!?), but
    • When using -Weverything, DO whitelist:
      • -Wno-c++89-compat[-pedantic] : we make agressive use of C++11 and don't care at all about C++98 compatibility
      • -Wno-global-constructors, -Wno-exit-time-destructors : just for now
      • -Wpadded : hmmm... why?
  1. Use override for virtual method overrides.

    • If you think you're overriding a method, tell the compiler; the compiler will tell you if you're mistaken (e.g. the superclass declaration changes):

      {% highlight c++ %} class Base { // howSilly recently changed from int to size_t virtual bool sillyEnough(size_t howSilly) const = 0; };

      class Derived { // out of sync with Base; causes a compiler error rather // than incorrect runtime behaviour bool sillyEnough(int howSilly) const override; }; {% endhighlight %}

Preferences

These things aren't requirements, but we consistency is good, so I've made some choices. Some of these choices will seem familiar if you know the Qt style guide, but others might be fairly arbitrary. We could change these things, but changing lots of code should only be done for a good reason.

  1. Name simple accessors simply : Foo::parent(), not Foo::getParent()
  1. Give member variables an underscore suffix : the most sensible name for a member variable is probably also the best name for its accessors, so give the variable an underscore (and this will also help with -Wshadow)
