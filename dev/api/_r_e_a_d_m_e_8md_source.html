1 fabrique
2 ========
3 Fabrique is a *meta* build tool.
4 Like other such tools (CMake, SCons...) it is the first in a two-step process:
5 it configures your project's build, defined in a programming language designed
6 for the purpose and generates a build file (Makefile, Ninja file...).
7 The build file is written in a language designed for *that* purpose, and in
8 some cases (e.g. Ninja) it does it rather well.
9 All meta build tools recognise that simply adding more syntax to Make
10 is not a readable, portable way to describe complex software.
11 
12 *Unlike* other meta build tools:
13 
14  * Fabrique has very few dependencies
15  * Fabrique is intended to be an alternative build system for FreeBSD
16  * FreeBSD's bootstrap environment does not include Java, M4 or Python
17  * it *does* include modern C++ tools (Clang and libc++), plus flex and byacc
18  * Fabrique is designed for ease of readability and build maintenance
19  * types, not macros
20  * macro expansion is a neat trick for small scripts, but:
21  * some CMake functions expect "a b c"; others expect "a;b;c" (!?)
22  * how many escapes do I need in this Make-expanded inline shell script?
23  * large-scale software is held together by *types* and *interfaces*
24  * consistency, not context-dependence
25  * no "black magic":
26  * `CFLAGS=-DFOO make` vs `make CFLAGS=-DFOO`)
27  * `make -V CFLAGS` vs `make -V .IMPSRC`
28  * all configuration values are immutable and queryable
29  * explicit configuration enables reproducible builds
30  * functional, not imperative
31  * builds should be parallel
32  * build descriptions should not include:
33  * iterative loops
34  * time-varying variables
